guitar

django package manager

30 Sep, 2013: Current state of project is pre-pre-alpha. We are happy to hear your feedback or suggestions.

What?

guitar is a tool, which should help integrate existing django applications into your project code.

As you may know, installing some of applications is not so trivial as adding them into INSTALLED_APPS section of your settings.py file. You should go through application documentation, determine suitable settings and then put them inside your settings file. In addition you may need to update urls.py file, put middleware into proper order (after A, but before B, or so).

guitar trying simplify this process, by providing friendly commandline utility. Asking questions regarding different application options it will be able to create proper configuration and patch your project files, so that you no need to do this by hand.

Instalation

> pip install guitar Will install guital commandline utility and all it dependencies. In case you use virtualenv, you no need to have copy of guitar in each of your virtual environments. So that it can be installed system-wide.

Usage

> guitar search <package> Will perform searching of configurations (chord) for required package. As this is extremely young project, there not a lot of ready-to-use configurations, but this should be fixed in future.

If you want to help, you need to scroll to Contribute section.

> guitar install <package> Will check if package already installed (if no, it will be installed using pip), fetch configuration from server, ask you amount of questions and patch your project settings, urls files. You can tune/suggest guiter by calling with specific options.
You can see list of all available options by issuing > guitar -h.

Contribute

> guitar create <package> Will create barebone configuration file, which you need to tune. After you finish it you can contribute it, by sending pull request to repository with configurations.
